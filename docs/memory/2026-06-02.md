## Qué se hizo (sesión mañana)

### Google Reviews — sincronización y orden

- Investigado el sistema de Google My Business: OAuth2 + sync a tabla `google_reviews` en PostgreSQL.
- El flujo para actualizar reseñas es:
  1. `GET /api/google-mybusiness/auth-status` — verificar si el token sigue válido
  2. Si `needsReauth: true`: `GET /api/google-mybusiness/auth/start` (JWT requerido) → abrir `authUrl` en browser → Google redirige al callback
  3. `POST /api/google-mybusiness/reviews/sync` (JWT requerido) — fetch desde Google + guarda en DB
- La sync se hizo exitosamente desde Postman; las reseñas de mayo 2026 ya aparecen.
- El orden (DESC por `google_create_time`) ya estaba correcto en el backend. El problema de orden era en el frontend — resuelto del lado del front.

### Fix: `dev:demo` no levantaba el servidor

**Causa raíz:** `dev:demo` usaba `NODE_ENV=test`. El guard `if (NODE_ENV !== 'test')` en `app.ts` bloqueaba el `app.listen()`. Además, `.env.test` tenía `NODE_ENV=test` y `dotenv.config({ override: true })` lo repisaba incluso al intentar cambiar el script.

**Fix aplicado:**
- `package.json`: `dev:demo` cambia `NODE_ENV=test` → `NODE_ENV=demo`
- `src/app.ts`: env file logic incluye `demo` → carga `.env.test`; logs/morgan también incluyen `demo`
- `.env.test`: eliminada la línea `NODE_ENV=test` (NODE_ENV lo controla el script, no el archivo de entorno)
