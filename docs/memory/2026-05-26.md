- **Modelo pay-per-feature**: cada feature vive en su rama `feature/xxx`, va a `development` con tests, y se mergea a `main` solo cuando el cliente paga
- Flujo: `feature/xxx` → `development` (staging) → `main` (producción, cliente habilitado)
- Se mergeó `development` → `main` para alinear ambas ramas al mismo commit base
- Se eliminaron todas las ramas stale excepto `development`
