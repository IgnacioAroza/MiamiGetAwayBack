### Bugs encontrados y corregidos

**1. Equipaje siempre guardado como 0**

Causa: el controller `createInquiry` armaba el objeto `data` manualmente desde `req.body` y no incluía `luggage_large`, `luggage_medium`, `luggage_carry_on`. Zod los recibía como `undefined` y aplicaba el default `0`.

Fix: agregar los 3 campos al objeto `data` en el controller antes de llamar a `validateInquiry`.

**2. Loop infinito de requests al front (GET /api/transfers/vehicles 429)**

Causa frontend: `useEffect` con condición `items.length === 0` en el dep array. Cuando la API devuelve array vacío o falla, el efecto se re-ejecuta indefinidamente. El front lo corrigió en 3 archivos quitando la condición y dejando solo `[dispatch]`.

Causa backend: `/api/transfers` tenía `uploadRateLimit` (30 req/15min), demasiado restrictivo para GETs públicos. Corregido removiendo `uploadRateLimit` del mount — `globalRateLimit` (200 req/15min) ya los cubre.

**Nota importante descubierta:** cuando se para el servidor de Vite, el browser tab sigue ejecutando la app React en memoria. El proceso de Node/tsx en el puerto 3001 puede quedar huérfano cuando se cierra la terminal sin hacer Ctrl+C.

**3. Formato de hora en email**

El `time` se guardaba como `HH:mm` (24h) y llegaba al mail sin conversión. Agregado helper privado `formatTime12h(time: string)` en `EmailService` que convierte a formato 12h con AM/PM.
